Translate Keywords Country Export Documentation

Description

This tool allows users to translate their EN mailers to other CLE languages in-place on stripo itself.
It allows for undo of most recent translation.
It includes CRUD operations for the translations.json file that contains the key and values for translation.
Adding keywords will be done via text input and Updating and Deleting will be done via a dropdown of the regex keywords that is being replaced/matched.
Highlighting selected keyword is available after text input or selection from dropdown.
Tool also allows for users to export their local translations.json.
NOTE: Recommended workflow is for users to overwrite the translations.json in the folder that contains the extension BEFORE distributing to other users to prevent any problems.

Features

  1. Translate from EN to other CLE languages + Undo translation
  2. Add new Keywords and Replacements for other CLE languages
  3. Update existing keywords
  4. Delete existing keywords
  5. Download/Copy translations.json -> distribution

How to use

  1. User wants to translate -> just go to translate
  2. User finds that there is a keyword/phrase that they would want to add
    1. Go to keywords tab
    2. Type in the keyword they are looking to change
    3. Press highlight and check the page to see the affected areas
    4. Decides to pursue the translation and inputs the other translations into the fields and adds into the localstorage
    5. Consensus to not have a powerful new keyword for eg. 'for' / any single word
  3. User finds that he has made an error with the previous addition and would like to change it without accessing codebase
    1. Go to keywords tab
    2. Type or Selects from dropdown of keys for relevant keyword
    3. Checks through the populated input fields for errors such as spelling
    4. Change and update to json
    5. Check the affected area by selecting a language to change to and click test
    6. Once tested, clear the changes and then move on to translate or perform other tasks
    7. Another consensus that updates are only done for minor mistakes like spelling/update to more accurate foreign language translation
  4. User has made changes and would like to save changes on filesys
    1. Click on export
    2. Click on copy/download
    3. Open JSON file in text editor and replace contents with copied clipboard / download and replace the current translations.json
  5. User distributes and passes its full zip folder/translations.json file to another user
    1. User exports the file and saves it on local before pushing to github for version control
    2. User passes it through messaging channels
    3. New user will follow instructions to developer mode and then drag the zip file into extension manager page
